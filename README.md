# Holy Land Photos

A photography and biblical scholarship website by Dr. Carl Rasmussen, rebuilt with modern web technology.

**Live dev site:** https://holy-land-photos.vercel.app
**CMS admin:** https://holy-land-photos.vercel.app/admin
**Original site:** https://holylandphotos.org

## Tech Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| Framework | [Next.js](https://nextjs.org/) 15 | React-based web framework |
| CMS | [Payload CMS](https://payloadcms.com/) 3.0 | Content management, embedded in the Next.js app |
| Database | [Turso](https://turso.tech/) (libSQL/SQLite) | Remote SQLite database |
| Image CDN | `img.holylandphotos.org` | Existing image hosting with on-the-fly resizing |
| Photo Storage | [AWS S3](https://aws.amazon.com/s3/) | New photo uploads (dev bucket) |
| Hosting | [Vercel](https://vercel.com/) | Serverless deployment |

## Content

- **730 sections** organized in a 5-level hierarchy (Top Level > Country > Region > Site > Artifact)
- **7,025 photos** with scholarly descriptions, keywords, and metadata
- **36 static pages** (About, Permissions, How to Use, etc.)
- **171 news articles**
- **120 Site of the Week** entries

## Getting Started

### Prerequisites

- Node.js 18+
- npm
- [Turso CLI](https://docs.turso.tech/cli/installation) (for database management)

### Setup

```bash
# Clone the repo
git clone https://github.com/rasm0225/holy-land-photos.git
cd holy-land-photos

# Install dependencies
npm install

# Create .env file with required environment variables (see below)

# Start dev server
npm run dev
```

Visit `http://localhost:3000/admin` to access the CMS, or `http://localhost:3000` for the public site.

### Environment Variables

Create a `.env` file in the project root:

```
# Turso Database
DATABASE_URL=libsql://your-database-url.turso.io
DATABASE_AUTH_TOKEN=your-turso-auth-token

# Payload CMS
PAYLOAD_SECRET=a-random-secret-string

# AWS S3 (for new photo uploads)
S3_BUCKET=your-bucket-name
S3_ACCESS_KEY_ID=your-access-key
S3_SECRET_ACCESS_KEY=your-secret-key
S3_REGION=us-east-2
```

## Project Structure

```
holy-land-photos/
  payload.config.ts          # CMS configuration (collections, plugins, database)
  src/
    app/
      (frontend)/            # Public website pages
        page.tsx             # Homepage
        browse/[slug]/       # Section pages
        photos/[imageId]/    # Individual photo pages
      (payload)/             # CMS admin panel (auto-generated by Payload)
        admin/               # Admin UI routes
        api/                 # REST API routes
    collections/             # Payload collection definitions
      Sections.ts            # Site hierarchy (730 sections)
      Photos.ts              # Photo metadata (7,025 photos)
      Pages.ts               # Static pages
      News.ts                # News articles
      SiteOfTheWeek.ts       # Featured sites
      SectionPhotos.ts       # Junction table (hidden, pending removal)
      Users.ts               # Admin users
    components/
      KeywordTagInput/       # Custom tag chip input for keywords
      SectionHierarchy/      # Custom admin tree view of all sections
      KeywordIndex/          # Custom admin keyword browser with filter
  scripts/
    migrate.py               # CSV-to-Turso migration script
  mockups/                   # Static HTML mockups (design reference)
  docs/                      # Documentation and reference files
```

## CMS Admin Features

- **Section Hierarchy** — tree view of all 730 sections at `/admin/section-hierarchy`
- **Keyword Index** — searchable index of all keywords at `/admin/keyword-index`
- **Tag chip input** — comma-separated keyword entry with removable pills
- **Auto-generated slugs** — section and page slugs derived from titles
- **Photo management** — drag-and-drop photo ordering within sections
- **Nested sections** — parent/child hierarchy with auto-generated breadcrumbs

## Public Frontend

Unstyled functional pages (styling is a future phase):

- `/` — homepage with top-level section links
- `/browse/{slug}` — section page with body text, child sections, and photo grid
- `/photos/{imageId}` — individual photo with description and linked sections

Photos are served from the existing `img.holylandphotos.org` CDN.

## Scripts

### Data Migration

```bash
# Import legacy CSV data into Turso
python3 scripts/migrate.py /path/to/Archive
```

The migration script imports sections, photos, photo-section links, pages, news, and site of the week entries from CSV exports of the original ASP/SQL Server database.

### Payload Commands

```bash
npm run dev                    # Start dev server
npm run build                  # Production build
npx payload generate:types     # Regenerate TypeScript types
npx payload generate:importmap # Regenerate admin import map (after adding components)
```

## Deployment

The site deploys to Vercel. Currently deployed manually:

```bash
npx vercel --prod
```
